

# 记忆服务API接口文档

**39.104.81.41换成互联网地址39.104.81.41**

**基础URL**: `http://39.104.81.41:8001`  
**服务描述**: 提供用户记忆的存储和检索功能，支持个性化记忆管理

---

## 1. 记忆检索接口

### 1.1 接口描述
根据用户输入的问题，检索相关的用户记忆信息。

### 1.2 请求信息

| 项 | 说明 |
|---|---|
| **接口名称** | 记忆检索 |
| **URL** | `/msearch` |
| **方法** | `POST` |
| **Content-Type** | `application/json` |

### 1.3 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|---|---|---|---|
| sentence | string | 是 | 用户输入的查询语句 |
| userid | string | 是 | 用户唯一标识 |
| topk | integer | 是 | 返回的最相关记忆数量 |

### 1.4 请求示例

```bash
curl -X 'POST' \
  'http://39.104.81.41:8001/msearch' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
    "sentence": "定时任务有哪些",
    "userid": "001536031",
    "topk": 3
  }'
```

### 1.5 响应参数

| 参数名 | 类型 | 描述 |
|---|---|---|
| code | integer | 状态码，200表示成功 |
| details | string | 详细信息（成功时为空） |
| time | float | 处理耗时（秒） |

### 1.6 响应示例

```json
{
  "details": "",
  "time": 0.6538341045379639,
  "code": 200
}
```

## 2. 记忆存储接口

### 2.1 接口描述
存储用户的个性化记忆信息。

### 2.2 请求信息

| 项 | 说明 |
|---|---|
| **接口名称** | 记忆存储 |
| **URL** | `/madd` |
| **方法** | `POST` |
| **Content-Type** | `application/json` |

### 2.3 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|---|---|---|---|
| sentencein | string | 是 | 用户输入的原始信息 |
| sentenceout | string | 是 | 系统回复的信息 |
| userid | string | 是 | 用户唯一标识 |

### 2.4 请求示例

```bash
curl -X 'POST' \
  'http://39.104.81.41:8001/madd' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
    "sentencein": "我叫李建龙，生日是12月12号，我喜欢吃苹果和龙虾",
    "sentenceout": "好的知道了",
    "userid": "001536031"
  }'
```

### 2.5 响应参数

| 参数名 | 类型 | 描述 |
|---|---|---|
| code | integer | 状态码，200表示成功 |
| message | string | 响应信息 |
| userid | string | 用户唯一标识 |
| time | float | 处理耗时（秒） |

### 2.6 响应示例

```json
{
  "code": 200,
  "message": "用户记忆存储成功",
  "userid": "001536031",
  "time": 9.72280478477478
}
```

---

## 3. 错误码说明

| 状态码 | 说明 |
|---|---|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 404 | 接口不存在 |
| 500 | 服务器内部错误 |

---

## 4. 注意事项

1. **userid** 字段必须保持一致，用于区分不同用户的记忆
2. 记忆存储接口的 `sentencein` 和 `sentenceout` 需成对出现，确保对话上下文完整
3. 检索接口的 `topk` 参数建议设置在1-10之间，避免返回过多冗余信息
4. 所有时间字段 `time` 单位为秒
