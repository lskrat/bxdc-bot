## Backend (Skill-gateway) 功能模块详解

```

│   │   └── DELETE /conversations/{id}（删除）
│   ├── 对话运行 API
│   │   └── POST /conversations/{id}/run（SSE 流）
│   ├── Skill 管理 API
│   │   ├── POST /skills（创建）
│   │   ├── GET /skills（列表）
│   │   ├── GET /skills/{id}（详情）
│   │   ├── PUT /skills/{id}（更新）
│   │   └── DELETE /skills/{id}（删除）
│   ├── 确认提交 API
│   │   └── POST /conversations/{id}/confirm
│   └── 内部接口（供 agent-core 调用）
│       ├── POST /internal/agent/run（代理）
│       ├── POST /internal/logs/conversation
│       └── POST /internal/logs/skill-invocation

├── 业务层 (Service)

│   │   ├── 确认请求生成
```

## 数据流向示意

```
用户操作 → Frontend
              ↓
         调用 API
              ↓
         Skill-gateway（接收）
              ├── 鉴权检查
              ├── 参数校验
              ├── 日志记录
              ↓
         业务处理
              ├── 对话管理：查/改数据库
              ├── Skill 执行：调用执行器
              └── Agent 代理：转发请求
              ↓
         Agent-core（仅内部访问）
              ├── ReAct 循环
              ├── LLM 交互
              └── 工具调用（回调 gateway）
              ↓
         Skill-gateway（响应）
              ├── 接收 SSE 流
              ├── 代理转发给 Frontend
              └── 异步记录日志
              ↓
         Frontend（展示）
              └── 流式渲染消息
```

## 模块间的核心交互

| 交互双方 | 交互方式 | 核心数据 |
|----------|----------|----------|
| Frontend ↔ Skill-gateway | HTTP / SSE | 对话消息、Skill 配置 |
| Skill-gateway ↔ Agent-core | HTTP / SSE | 用户输入、历史消息、可用 Skill 列表 |
| Skill-gateway ↔ MySQL | JDBC | 对话、消息、Skill、日志 |
| Skill-gateway ↔ 外部服务 | HTTP / SSH | Skill 执行请求 |
| Agent-core ↔ Skill-gateway (回调) | HTTP | 工具调用请求 |
