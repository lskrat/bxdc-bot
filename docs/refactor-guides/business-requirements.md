- **文档用途**：供产品经理、需求分析师、开发人员理解业务需求
- **维护方式**：需求变更时同步更新此文档

---

## 1. 产品概述

### 1.1 产品定位

AI Agent 对话平台，让用户通过自然语言与智能助手交互，调用预定义工具（Skill）完成各类任务。

### 1.2 目标用户

| 角色 | 主要职责 | 使用场景 |
|------|----------|----------|
| 业务人员 | 日常使用对话功能 | 查询数据、生成报告、执行操作 |
| Skill 管理员 | 管理 Skill | 创建、配置、发布 Skill |
| 运维人员 | 查看日志、排查问题 | 审计查询、故障排查 |

### 1.3 核心业务流程

```
用户登录 → 创建/选择对话 → 配置可用 Skill → 发送消息
                                            ↓
                                    AI 理解意图 → 调用 Skill → 返回结果
                                            ↓
                                    查看历史对话 → 继续使用或结束
```

#### 3.1.4 核心功能点

| 功能编号 | 功能名称 | 功能描述 | 优先级 |
|----------|----------|----------|--------|
| P01-F01 | 新建对话 | 创建新对话会话 | 高 |
| P01-F02 | 对话列表 | 展示用户所有对话，支持切换 | 高 |
| P01-F03 | 对话删除 | 删除不需要的对话 | 中 |
| P01-F04 | 消息发送 | 输入文本并发送 | 高 |
| P01-F05 | 流式接收 | 实时接收并展示 AI 回复 | 高 |

#### 3.1.5 字段说明

**对话列表项**：
| 字段 | 说明 | 示例 |
|------|------|------|
| 对话标题 | 自动生成或用户自定义 | "订单数据分析" |
| 最后更新时间 | 相对时间显示 | "刚刚" / "5分钟前" / "昨天" |
| 消息预览 | 最后一条消息前 20 字 | "帮我查一下昨天..." |

**消息展示**：
| 字段 | 说明 | 样式 |
|------|------|------|
| 用户消息 | 用户输入内容 | 右侧，蓝色背景 |
| AI 消息 | 助手回复内容 | 左侧，白色背景，支持 Markdown |
| 工具消息 | Skill 调用过程和结果 | 灰色，可折叠 |

**输入区域**：
| 字段 | 说明 | 限制 |
|------|------|------|
| 输入框 | 文本输入 | 最多 2000 字 |
| 发送按钮 | 触发消息发送 | 空内容禁用 |

#### 3.1.6 交互规则

**新建对话**：
- 点击"[+] 新建对话"按钮
- 系统创建新对话，标题默认为"新对话"（或按时间命名）
- 自动切换到新对话，清空聊天区域

**切换对话**：
- 点击左侧对话列表中的某一项
- 加载该对话的历史消息（最近 20 条）
- 保持当前对话的输入框内容不变

**发送消息**：
- 在输入框输入文本
- 点击发送按钮或按 Enter 键
- 消息出现在聊天区域（用户侧）
- 等待 AI 回复（展示加载状态）
- 收到 SSE 流，逐字展示回复内容

**历史加载**：
- 向上滚动到顶部
- 自动加载更早的 20 条消息
- 保持滚动位置

**Skill 配置**：
- 点击聊天区域上方的"Skill 配置"按钮
- 弹出 Skill 选择弹窗
- 勾选当前对话要启用的 Skill（多选）
- 保存后生效

#### 3.1.7 异常处理

| 异常情况 | 处理方式 |
|----------|----------|
| 网络断开 | 提示"网络异常，请检查连接"，提供重试按钮 |
| 发送失败 | 消息旁边显示红色感叹号，点击重发 |
| AI 超时 | 提示"响应超时，请稍后重试" |
| 无可用 Skill | 提示"当前对话未配置 Skill，请先配置" |

#### 3.3.6 交互规则

- 点击类型单选按钮
- 下方动态表单区域切换为对应配置
- 切换时保留已填的通用字段（名称、描述、分类）

**参数配置**：
- 点击"添加参数"增加一行
- 每行包含：名称、类型（string/number/boolean）、必填勾选、默认值、描述
- 可拖拽排序或删除

**保存并测试**：
- 先执行保存
- 弹出测试抽屉，显示测试表单（根据参数配置生成）
- 用户填写参数，点击"执行"
- 展示执行结果

**保存**：
- 校验必填字段
- 校验失败时滚动到第一个错误项并提示
- 保存成功后返回列表页，新 Skill 置顶显示

#### 3.3.7 异常处理

| 异常情况 | 处理方式 |
|----------|----------|
| 名称重复 | 提示"Skill 名称已存在，请更换" |
| URL 格式错误 | 输入框变红，提示"请输入有效的 URL" |
| 测试执行失败 | 展示错误信息，帮助排查问题 |
| 参数名重复 | 提示"参数名称不能重复" |

### P04 Skill 编辑页面

#### 3.4.1 页面定位

修改已有 Skill 配置的表单页面，与新增页面结构相同，预填充现有数据。

#### 3.4.2 页面入口

#### 3.4.3 与新增页面的差异

```

                                   查看调用链
                                         ↓
```

### 6.2 状态定义

**对话状态**：
- active：正常可用
- archived：已归档（不在默认列表显示）
- deleted：已删除（软删除，保留数据）

**Skill 来源**：
- builtin：系统内置
- extension：用户自建
- public：其他用户公开

**Skill 类型**：
- api：调用 HTTP 接口
- ssh：执行 SSH 命令
- template：提示词模板
- openclaw：自主编排
