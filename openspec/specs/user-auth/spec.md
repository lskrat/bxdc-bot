# user-auth

## Purpose

定义用户注册、登录、会话持久化及用户切换的认证流程。

## Requirements

### Requirement: 用户注册

系统 MUST 提供新用户注册机制，包括生成头像的步骤；**注册前 MUST 校验调用方提供的系统管理员密码（`systemAdminPassword`）与部署配置 `app.registration.admin-password` 一致，校验通过后方可创建用户**；未通过校验时 MUST NOT 创建用户记录。

#### Scenario: 成功注册

- **WHEN** 用户提交有效的 6 位数字 ID 和昵称（1–10 个字符）
- **AND** 用户提交了与服务器配置的**系统管理员密码**一致的凭据
- **THEN** 系统创建一个新用户记录
- **AND** 显示「正在生成头像...」状态
- **AND** 调用 AI 生成头像
- **AND** 生成完成后显示「进入对话」按钮
- **AND** 用户点击后登录并进入对话界面

#### Scenario: 管理员凭据未通过

- **WHEN** 用户提交注册请求但系统管理员密码缺失、错误或与服务端配置不一致
- **THEN** 系统拒绝注册
- **AND** MUST NOT 创建用户记录
- **AND** 向用户展示明确但不过分泄露内部判因的错误提示（例如无注册权限或凭据无效）

#### Scenario: 重复 ID 注册

- **WHEN** 用户提交一个已存在的 ID（仅在管理员凭据通过时才会走到本检查）
- **THEN** 系统拒绝注册
- **AND** 显示错误消息「ID 已存在」

#### Scenario: 无效输入

- **WHEN** 用户提交无效的 ID（非 6 位数字）或昵称（为空或超过 10 个字符）
- **THEN** 系统拒绝注册并显示验证错误

### Requirement:用户登录

系统 MUST 允许现有用户使用其 ID 登录。

#### Scenario: 成功登录

- **当** 用户输入已注册的 6 位数字 ID
- **那么** 系统验证用户身份
- **并且** 恢复其个人资料和头像
- **并且** 加载其特定的记忆上下文

#### Scenario: 登录失败

- **当** 用户输入未注册的 ID
- **那么** 系统显示错误「用户不存在」

### Requirement:会话持久化

系统 MUST 在浏览器会话之间持久化用户的登录状态。

#### Scenario: 浏览器刷新

- **当** 已登录用户刷新页面
- **那么** 他们保持登录状态
- **并且** 其上下文得到保留

### Requirement:用户切换

系统 MUST 允许在不同用户账户之间切换。

#### Scenario: 切换用户

- **当** 用户选择「切换用户」或「退出」
- **那么** 清除当前会话
- **并且** 用户返回登录/注册屏幕
