## Requirements
### Requirement: 用户独立服务器台账
系统 MUST 为每个登录用户维护独立的服务器台账，在**同一张**业务表中将服务器**展示/检索用** 名称、网络可达地址、登录名与**认证信息** 与台账主键一并持久化在 Java 数据库中，并作为通过 **`X-User-Id` + 台账主键** 在服务端执行 Linux 脚本的**凭据**来源，与 `Linux 脚本以台账为凭据主数据源` 要求一致。

#### Scenario: 创建当前用户服务器台账
- **WHEN** 已登录用户提交合法的网络地址、用户名、**密码**（或经任务定义的其他认证方式）及**维护接口所要求的** 展示/检索用名称等字段
- **THEN** 系统创建一条归属于该用户的服务器台账记录
- **AND** 该记录被持久化保存

#### Scenario: 用户之间台账隔离
- **WHEN** 用户 A 查询服务器台账列表
- **THEN** 系统只返回归属于用户 A 的记录
- **AND** 不返回用户 B 的任何服务器台账

### Requirement: 服务器台账维护接口
系统 MUST 提供当前用户维度的服务器台账查看、新增、编辑、删除能力。

#### Scenario: 查看当前用户台账列表
- **WHEN** 已登录用户请求查看服务器台账
- **THEN** 系统返回该用户的服务器列表
- **AND** 每条记录至少包含可识别的 `ip` 和 `username`

#### Scenario: 编辑服务器台账
- **WHEN** 已登录用户更新自己的一条服务器台账记录
- **THEN** 系统保存新的字段值
- **AND** 后续查询返回更新后的结果

#### Scenario: 删除服务器台账
- **WHEN** 已登录用户删除自己的一条服务器台账记录
- **THEN** 系统移除该记录
- **AND** 该记录不再出现在后续列表查询中

### Requirement: 同一用户 IP 唯一性
系统 MUST 阻止同一用户重复维护相同 `ip` 的服务器台账，以避免 SSH 解析出现歧义。

#### Scenario: 重复创建相同 IP
- **WHEN** 已登录用户新增一条 `ip` 与其现有台账重复的服务器记录
- **THEN** 系统拒绝创建
- **AND** 返回明确的重复错误信息

### Requirement: 台账访问控制
系统 MUST 阻止用户访问或修改不属于自己的服务器台账记录。

#### Scenario: 越权访问他人记录
- **WHEN** 已登录用户请求查看、编辑或删除不属于自己的服务器台账
- **THEN** 系统拒绝该请求
- **AND** 不泄露目标记录的详细信息

### Requirement: 列表接口最小化返回敏感字段
系统 MUST 避免在服务器台账列表接口中返回明文密码。

#### Scenario: 查看列表时隐藏密码
- **WHEN** 已登录用户查看自己的服务器台账列表
- **THEN** 系统不在列表响应中返回明文 `password`
- **AND** 用户仍可基于列表结果选择要维护的服务器记录

### Requirement: Linux 脚本以台账为凭据主数据源
系统 MUST 将 Linux 脚本（或等价的远程命令执行）的目标连接与认证信息，限定为从**同一张**服务器台账表中、由 `userId` 与台账主键 `id` 唯一标识的行内读取，而**不** 将应用级环境配置中按名称映射的 `skill.linux-script.servers.*` 等作为**生产环境**的**权威**凭据来源（开发或迁移回退若存在，MUST 在实现上显式可关闭，且**不得**在默认安全模型下与台账行冲突）。

#### Scenario: 仅台账行可驱动执行
- **WHEN** 已认证请求携带**有效**的台账 `id` 与要执行的 `command`
- **THEN** 系统**仅**通过当前用户下该 `id` 的台账行获取 `ip`（或 `host`）、`username`、`password`（及 `port` 等可选字段）
- **AND** 成功建立连接**当且仅当**该台账行**满足**可连接所需的最小数据完整性
- **AND** 不将「环境键与台账 `name` 对齐」作为**主路径**的连机前提
