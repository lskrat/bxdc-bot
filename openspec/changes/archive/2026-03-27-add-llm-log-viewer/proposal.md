## Why

当前系统虽然会把大模型交互写入 `llm.log`，但日志是面向开发者的原始文本追加记录，用户无法在当前对话中直接查看，也难以区分“发给大模型的参数”与“大模型返回的内容”。随着对 Agent 行为可解释性和调试效率的要求提高，需要把这部分日志结构化，并提供一个可在聊天界面实时查看的日志入口。

## What Changes

- 新增面向单次对话/任务的结构化 LLM 日志能力，记录发送给大模型的请求参数与大模型返回内容。
- 为聊天界面增加“日志查看”入口，支持在当前对话中打开并查看该轮会话的 LLM 调用日志。
- 将日志内容从纯文本写盘升级为可结构化消费的数据模型，明确区分请求侧与响应侧字段。
- 支持日志查看器根据当前对话的流式进展实时更新，而不是只在会话结束后一次性展示。
- 保留落盘能力，但要求结构化日志能够映射回日志文件与前端展示视图，便于排查。

## Capabilities

### New Capabilities
- `llm-log-viewer`: 定义当前对话的结构化 LLM 请求/响应日志采集、实时更新与查看行为。

### Modified Capabilities
- `chat-ui`: 聊天界面需要增加日志查看按钮与日志面板/弹窗入口，并在当前回复过程中持续展示更新后的日志内容。
- `agent-client`: 前端客户端需要接收或拉取结构化 LLM 日志数据，并将其与当前对话会话关联。
- `api-gateway`: 任务相关接口/事件流需要向前端暴露可消费的结构化 LLM 日志更新。

## Impact

- 影响 `backend/agent-core` 的 LLM 日志采集、结构化模型、日志写入与会话关联。
- 影响任务事件流或配套日志接口，需要把日志增量安全地暴露给前端。
- 影响前端聊天界面的工具栏/消息区，新增日志查看按钮与结构化日志展示面板。
- 影响前端响应式状态管理，需要按当前对话实时维护请求参数与模型响应的日志列表。
- 需要补充结构化日志格式、隐私边界与实时更新行为的验证。
