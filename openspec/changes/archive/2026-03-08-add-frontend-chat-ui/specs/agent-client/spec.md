## 新增需求

### 需求：任务提交
客户端必须通过 HTTP POST 将用户消息发送到后端。

#### 场景：发送任务
- **当**用户提交消息时
- **那么**客户端发送带有消息内容的 `POST /api/tasks` 请求
- **并且**处理响应（成功或错误）

### 需求：实时更新
客户端必须订阅 Server-Sent Events (SSE) 以获取 Agent 更新。

#### 场景：订阅事件
- **当**任务成功提交时
- **那么**客户端打开到 `/api/tasks/{id}/events` 的 `EventSource` 连接
- **并且**监听 `message` 事件
- **并且**根据事件数据更新 UI

### 需求：连接处理
客户端必须处理连接错误和重连。

#### 场景：连接断开
- **当** SSE 连接中断时
- **那么**客户端尝试自动重连
- **并且**如果重连失败则通知用户
