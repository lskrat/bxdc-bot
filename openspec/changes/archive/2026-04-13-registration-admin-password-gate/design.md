## Context

当前 Fishtank 前端提供开放注册：任意访客可提交 6 位 ID 与昵称并创建用户。需改为「邀请制/门禁制」：仅持有**系统管理员密码**的访客才能完成注册。产品给定口令 **`Bxdc1357`**，按需求写死在应用中（前后端一致规则见下）。

## Goals / Non-Goals

**Goals:**

- 未验证管理员凭据前，后端 **拒绝** 创建用户（关闭开放注册）。
- 注册请求携带管理员密码字段，由服务端校验成功后，再走既有用户创建、头像等流程。
- 登录、会话、切换用户等行为保持与现 spec 一致。

**Non-Goals:**

- 多管理员、轮换口令、审计日志完备建模（可后续迭代）。
- 将口令改为环境变量或 KMS（本 change 按产品要求写死；design 记录后续迁移方向）。

## Decisions

1. **校验主体在后端**  
   仅前端隐藏注册入口不够；`POST`（或等价）注册接口 MUST 校验管理员密码，错误返回 401/403 及统一错误文案，不区分「口令错误」与「未提供」以降枚举风险（可选产品决定是否细分）。

2. **口令存放**  
   Java 侧使用常量或 `@Value` 默认值为 `Bxdc1357`（与需求一致）；**禁止**写入客户端构建产物以外的公开文档外的日志。前端：用户输入框收集口令，**不得**硬编码在可被爬虫索引的静态文案中——实现时在请求体发送即可；若必须在前端内置比对仅作 UX 预检，仍以服务端为准。

   > 说明：仓库内硬编码密码仅适合受控演示；生产环境应改为环境变量或秘钥服务。

3. **字段命名**  
   建议使用 `systemAdminPassword` 或 `adminGatePassword`（与现有 `UserCreateRequest` 对齐命名风格），与业务字段并列。

4. **时序比较**  
   服务端字符串比较使用恒定时间比较（如 `MessageDigest.isEqual` 包装 UTF-8 bytes），防计时旁路（低威胁场景仍建议）。

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 口令写入仓库泄露 | 限制仓库可见性；后续改为 `REGISTER_ADMIN_SECRET` 环境变量 |
| 前端仅存本地路由守卫可被绕过 | 以服务端校验为唯一权威 |
| 用户忘记口令无法自助注册 | 由运维/管理员分发口令（符合当前「门禁」定位） |

## Migration Plan

1. 部署新版本后端（校验生效）与前端（表单项上线）；旧客户端若不带字段 → 一律拒绝注册。
2. 回滚：恢复旧版本或 feature flag（若引入）恢复开放注册。

## Open Questions

1. 错误提示是否暴露「管理员密码错误」与「未填写」——默认建议泛化为「无注册权限」或等价，由产品定稿。
2. 是否在登录界面隐藏「注册」 tab 直至某本地开关——可与后端错误协同，非必须。
