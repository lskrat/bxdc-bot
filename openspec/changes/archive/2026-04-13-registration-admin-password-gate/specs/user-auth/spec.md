## MODIFIED Requirements

### Requirement: 用户注册

系统 MUST 提供新用户注册机制，包括生成头像的步骤；**注册前 MUST 校验调用方提供的系统管理员密码与部署配置的约定口令一致，校验通过后方可创建用户**；未通过校验时 MUST NOT 创建用户记录。

#### Scenario: 成功注册

- **WHEN** 用户提交有效的 6 位数字 ID 和昵称（1–10 个字符）
- **AND** 用户提交了与服务器配置的**系统管理员密码**一致的凭据
- **那么** 系统创建一个新用户记录
- **并且** 显示「正在生成头像...」状态
- **并且** 调用 AI 生成头像
- **并且** 生成完成后显示「进入对话」按钮
- **并且** 用户点击后登录并进入对话界面

#### Scenario: 管理员凭据未通过

- **WHEN** 用户提交注册请求但系统管理员密码缺失、错误或与服务端配置不一致
- **那么** 系统拒绝注册
- **并且** MUST NOT 创建用户记录
- **并且** 向用户展示明确但不过分泄露内部判因的错误提示（例如无注册权限或凭据无效）

#### Scenario: 重复 ID 注册

- **当** 用户提交一个已存在的 ID（且若实施管理员校验，则仅在管理员凭据也通过时才会走到重复检查）
- **那么** 系统拒绝注册
- **并且** 显示错误消息「ID 已存在」

#### Scenario: 无效输入

- **当** 用户提交无效的 ID（非 6 位数字）或昵称（为空或超过 10 个字符）
- **那么** 系统拒绝注册并显示验证错误
