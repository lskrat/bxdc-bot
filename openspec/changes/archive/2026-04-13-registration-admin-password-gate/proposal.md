## Why

开放注册允许任意访客创建账号，在演示或小团队部署中需要收敛入口：仅在被授予「系统管理员密码」凭据后，才允许完成注册流程，避免未授权开户。

## What Changes

- **关闭开放注册**：未通过管理员验证时，不得创建新用户（注册 API / 流程拒绝创建用户）。
- **管理员密码门禁**：注册前须验证一次「系统管理员密码」；校验通过后才进入现有注册字段（6 位 ID、昵称、头像等）流程。
- **内置管理员口令**：产品按需求固定为 **`Bxdc1357`**（由服务端与前端一致校验入口；见 design 安全与运维说明）。
- **BREAKING**：对现有一键注册行为不兼容——未提供正确管理员密码将无法注册（登录不受影响）。

## Capabilities

### New Capabilities

- （本变更以修改既有用户认证行为为主，不新增独立 capability 目录；若团队偏好单独拆分，可再设 `registration-admin-gate/` 并引用 `user-auth`。）

### Modified Capabilities

- `user-auth`: 注册流程增加管理员密码步骤；明确「非验证不得注册」与错误表现；登录与会话要求不变。

## Impact

- **后端**：`skill-gateway` 用户注册接口（及若有独立 auth 服务）增加管理员口令校验；配置或常量中声明预期口令。
- **前端**：注册 UI 增加管理员密码输入；请求体或预检步骤携带该字段；错误提示与禁用态。
- **运维**：管理员口令写死在仓库中，仅适合受信环境；生产应改用密钥管理或环境变量（可后续变更）。
