## MODIFIED Requirements

### Requirement: 用户独立服务器台账
系统 MUST 为每个登录用户维护独立的服务器台账，在**同一张**业务表中将服务器**展示/检索用** 名称、网络可达地址、登录名与**认证信息** 与台账主键一并持久化在 Java 数据库中，并作为通过 **`X-User-Id` + 台账主键** 在服务端执行 Linux 脚本的**凭据**来源，与 `Linux 脚本以台账为凭据主数据源` 要求一致。

#### Scenario: 创建当前用户服务器台账
- **WHEN** 已登录用户提交合法的网络地址、用户名、**密码**（或经任务定义的其他认证方式）及**维护接口所要求的** 展示/检索用名称等字段
- **THEN** 系统创建一条归属于该用户的服务器台账记录
- **AND** 该记录被持久化保存

#### Scenario: 用户之间台账隔离
- **WHEN** 用户 A 查询服务器台账列表
- **THEN** 系统只返回归属于用户 A 的记录
- **AND** 不返回用户 B 的任何服务器台账
