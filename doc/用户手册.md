# 用户手册：技能（Skill）与扩展能力

本手册面向**普通用户**，说明本项目中「技能」是什么、有哪些类型，以及如何通过**界面**或**对话里的技能生成工具**新增、修改扩展技能。

---

## 0. 开始使用：账号与浏览器

### 浏览器要求

- 请使用 **Chrome 86 及以上版本**，或同版本 **Chromium 内核**的浏览器（部分厂商内置浏览器、桌面版微信内置页等若版本过低，可能出现按钮错位或样式异常）。
- 若界面显示异常，请优先尝试**升级浏览器**后再试。

### 注册与登录

- **登录**：已注册用户输入 **6 位数字用户 ID** 即可进入应用。
- **注册**：本环境**不向公网开放自助注册**。创建新账户时，注册页会要求填写 **「系统管理员密码」**；该口令由**管理员单独提供**，请勿在公开场合传播。未填写或填写错误将无法创建账户。
- 若您没有管理员密码，请联系**单位/部署方的管理员**申请开户或索取授权。

## 1. 技能是什么？

在这个应用里，**技能**是助手可以使用的**能力模块**。例如：调用某个网站接口查数据、在服务器上执行只读命令、套用一段固定提示词，或按步骤串联使用多个工具。

技能分为两类：

| 类别 | 说明 |
|------|------|
| **内置技能（Built-in）** | 由系统提供，列表里可见但**不能**在「扩展技能管理」里删除或改配置。包括通用 API 调用、计算（Compute）、在指定 Linux 上执行脚本、**技能生成器**等。 |
| **扩展技能（Extended）** | 由您或管理员在 **Skill Gateway** 中登记的技能，可在 **Skill Hub → 管理** 中**新增、编辑、启用/停用、删除**。 |

您在对话中使用的「工具」，很多就来自这些技能（内置 + 已启用的扩展）。

---

## 2. 从哪里打开技能相关界面？

1. 打开应用侧栏或聊天界面上的 **Skill Hub**（技能中心，具体按钮名称以界面为准）。
2. 在 **Skill Hub** 抽屉中可以看到：
   - **Built-in Skills**：内置能力说明；
   - **Extended Skills**：已注册的扩展技能列表。
3. 点击 **「管理」**，打开 **「Extended Skill 管理」** 窗口，即可：
   - **新增 Skill**；
   - 在表格中 **编辑**（笔形图标）、**删除**、或用开关 **启用/停用**。

> **提示**：扩展技能保存在服务端（Skill Gateway）。请保证前端配置的网关地址正确，否则列表可能加载失败。

---

## 3. 两种「执行模式」要先选对

在新增或编辑技能时，需要先选择 **Execution Mode（执行模式）**：

| 模式 | 界面名称 | 适合做什么 |
|------|-----------|------------|
| **预配置** | `CONFIG` | 每一项能力都**事先写好**（调哪个网址、执行哪条命令、或固定提示词）。助手按配置执行，行为相对可预期。 |
| **自主规划** | `OPENCLAW` | 给助手一段**总说明**和一份**允许使用的工具名单**，由模型在限制范围内**自己决定**先调哪个工具、调几次。适合「多步任务」但**不保证**步骤顺序 100% 固定。 |

- 要选 **API / SSH / 模板** 三种「基础类型」时，请使用 **预配置**。
- **自主规划** 仅对应 **openclaw** 这一类配置（见下文）。

---

## 4. 预配置模式下的三种基础类型

在 **预配置** 下，通过 **「基础类型」** 选择：

### 4.1 API（调用 HTTP 接口）

**用途**：让助手通过统一网关去访问您填写的网址（GET/POST 等），常用于查天气、新闻、股票、内部接口等。

**建议填写**：

- **操作标识**：给这条能力起个内部名字（如 `juhe-joke-list`），便于区分。
- **请求方法**：如 `GET`、`POST`。
- **请求地址**：完整 URL（含协议，如 `https://...`）。
- **预配置模板**：
  - **通用接口**：自行填写地址与参数；
  - **当前时间**：可选用内置模板，快速对接示例时间接口（一般用户可照模板改）。
- **接口说明**（建议认真写）：用自然语言说明接口做什么、参数含义、返回里哪些字段重要。助手和用户都更容易理解。
- **参数格式契约（JSON）**（API 类**强烈建议**填写）：用 JSON Schema 描述助手调用时应传入的 JSON 字段（如必填 `gid`、`key` 等）。可减少乱传参数、便于校验。
- **Headers / Query / Body（JSON，可选）**：固定不变的请求头、查询参数或请求体写在这里；**每次对话里可变的参数**一般由助手按「契约」放在调用 JSON 的对应字段中（具体规则由系统合并，您只需知道：**契约里写什么名，助手就应传什么名**）。

**启用与隐私**：若 Query 里写了 API Key 等密钥，请仅在内网或可信环境使用，并限制谁能打开「管理」界面。

---

### 4.2 SSH（服务器巡检类命令）

**用途**：在已对接「服务器台账」的前提下，让助手在**指定机器**上执行您配置好的 **Shell 命令**（例如查看 CPU、内存、磁盘、端口等）。通常用于运维巡检类场景。

**说明**：

- 界面会固定一类 **预配置模板**（服务器状态巡检），包含操作标识、查找器、执行器等字段；**一般保持默认即可**，除非管理员另有要求。
- **执行命令**是核心：只建议配置**只读、安全**的命令；勾选 **只读模式** 表示该技能意图上不允许危险操作（具体仍以网关安全策略为准）。
- 实际执行时，助手往往还需要提供**服务器标识**（如名称或主机），这与您账户下的服务器登记有关。

若您不熟悉 Linux 命令，请让管理员代为配置，**不要随意粘贴网上复制的长命令**。

---

### 4.3 模板（可复用提示词）

**用途**：把一段**固定提示词**存成技能。助手调用时会把这段文字交给模型，用于翻译、格式化、按固定风格写作等**不需要联网接口**的任务。

**填写**：在 **提示词** 文本框中写入完整说明即可保存。

---

## 5. 自主规划（OPENCLAW）类型

当 **执行模式** 选 **自主规划** 时，配置的是 **openclaw** 技能：

- **提示词（Markdown）**：写清任务目标、步骤要求、注意事项（例如必须先取时间再计算日期差）。
- **允许工具列表**：填写助手**可以**调用的工具名称（如 `compute`、某个扩展技能的名称等）。**只填会真实存在的名字**，否则运行时会报错。
- **编排模式**：当前产品为 **serial（串行）**，即倾向于一步步调用，由模型在提示词约束下规划顺序。

**适合**：生日倒计时、需要先 A 再 B 的多步任务等。  
**注意**：顺序仍由模型理解提示词后决定，**不是**严格意义上的流程图引擎；若要绝对固定流水线，需要产品开发额外工作流能力。

---

## 6. 通用表单项：启用与需要确认

- **启用**：关闭后，该扩展技能不会在助手的工具列表中生效。
- **需要确认**：若开启，执行前可能需要用户确认（具体行为以当前版本实现为准），适合有一定风险的操作。

## 7. 用「技能生成工具」新增或更新技能（对话方式）

除在 **Skill Hub → 管理** 里手工填写外，助手还自带内置能力 **Skill Generator（技能生成器）**，工具名一般为 **`skill_generator`**。您可以用**自然语言**请助手帮您生成并注册扩展技能，例如：

> 「帮我加一个 API 技能：GET `https://api.example.com/data`，这是接口文档……参数有……」

### 7.1 生成器会做什么？

1. 按您提供的信息检查**必填项是否齐全**；
2. 向 Skill Gateway **创建**新技能；若同名已存在，需明确允许覆盖；
3. 对 **API 类型** 会尝试用默认或您提供的测试参数 **试调用一次**，把成功与否反馈在结果里。

### 7.2 各类型大致需要准备什么信息？

助手最终会以 **JSON** 调用生成工具。您不必自己写 JSON，但**心里要有数**缺什么助手会问您：

| 类型 | 在对话中尽量说清楚的内容 |
|------|-------------------------|
| **api** | 请求方法、完整 URL、**接口说明**（文档级描述）、**参数契约**（JSON Schema）。可选：Headers、Query、Body、测试用 `testInput`。 |
| **ssh** | 要在服务器上执行的**命令**（只读为佳）。 |
| **openclaw** | **系统提示词**（任务说明）；若要串联工具，说明**允许哪些工具**（名称需与系统一致）。 |
| **template** | 模板要用的**整段提示词**。 |

还可说明：**技能显示名称**、**简短描述**、是否**默认启用**、是否**需要确认**。

### 7.3 更新已有技能（重名）

若网关里**已有同名技能**，生成器会提示冲突。您需要让助手**带上允许覆盖**的选项（逻辑上对应 `allowOverwrite: true`），才会改为更新而不是报错。

### 7.4 生成之后还要做什么？

- 在 **Skill Hub** 里 **刷新**，确认新技能出现且已 **启用**。
- 若试调用失败，请根据返回说明检查 URL、密钥、参数契约是否与真实接口一致；也可在 **管理 → 编辑** 里微调后保存。

## 8. 界面操作速查：新增与编辑

### 新增

1. Skill Hub → **管理** → **新增 Skill**。  
2. 填 **名称**、**技能介绍**。  
3. 选择 **预配置** 或 **自主规划**。  
4. 预配置下再选 **基础类型**（API / SSH / 模板），按第 4 节填写。  
5. 勾选 **启用**（按需勾选 **需要确认**）→ 保存。

### 编辑

1. 在管理表格中点击该行的 **编辑**。  
2. 修改表单后保存。  

若系统提示**历史配置无法映射到表单**，说明该技能可能是旧格式或含界面暂不支持的字段，此时会显示**只读原始 Configuration**；请联系管理员或通过生成器/接口迁移为当前支持的写法。

### 删除

在表格行操作中确认 **删除** 即可（不可恢复，请谨慎）。

## 9. 常见问题（简要）

**问：注册时提示「暂无注册权限」或类似提示？**  
答：说明 **系统管理员密码** 未填写不正确或未获得授权。请向管理员确认当前环境是否允许新开账户，并**单独**获取口令，勿在群聊或邮件中公开发送。

**问：页面按钮点不动或布局混乱？**  
答：请先确认浏览器版本（见 **§0 浏览器要求**），并尝试刷新页面；仍异常时可反馈给管理员。

**问：为什么助手调接口总说参数不对？**  
答：检查 **参数格式契约** 是否与真实接口参数名一致；必填项是否都写在 `required` 里；固定密钥是否放在 **Query/Headers** 而不是让模型猜。

**问：内置的「技能生成」和界面「管理」有什么区别？**  
答：生成器适合**对话里快速起草并注册**；**管理**适合**精修字段、改名称、开关启用**，两者可配合使用。

**问：自主规划技能一定会按我写的 1、2、3 步执行吗？**  
答：不保证绝对顺序；模型会结合提示词与工具返回结果做决策。要硬性顺序需更强约束或未来工作流功能。
